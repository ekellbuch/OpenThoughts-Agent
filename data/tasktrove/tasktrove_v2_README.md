# TaskTrove

**TaskTrove** is an open-source collection of agentic task datasets, released by the [OpenThoughts-Agent](https://www.open-thoughts.ai/blog/agent) team. It is the task complement to **[AgentTrove](https://huggingface.co/datasets/open-thoughts/AgentTrove)** — the agent traces in AgentTrove were generated by running models against these task datasets using the [Harbor](https://github.com/open-thoughts/OpenThoughts-Agent) framework.

> **v2 (this release)** — 96 vetted, snapshot-safe, Harbor-format-valid task datasets, smoke-tested end-to-end and bucketed by difficulty. v1 is preserved at revision [`v1`](https://huggingface.co/datasets/open-thoughts/TaskTrove/tree/v1) for reproducibility of any prior runs.

---

## What changed in v2

- **Hard quality gate.** Every dataset was smoke-tested under a real Harbor agent; broken verifiers, snapshot-quota violations, and Harbor-format mismatches were patched before inclusion. Versions ending in `-v2`/`-v3`/`-v4`/`-v5`/`-v6` reflect those patches.
- **Difficulty buckets.** Each dataset is now tagged `easy` / `medium` / `hard` based on observed pass rates of a frontier teacher model (GLM-4.7 / Kimi K2.5 Thinking / GPT-5.x). Tasks with insufficient signal land in `unknown`.
- **Tighter scope.** v1's ~119 subdirs are replaced by the 96 vetted datasets below; superseded versions from v1 have been retired from `main` (still resolvable at the `v1` tag).
- **No format change.** Repository layout, task-binary format, and downstream pipeline (`extract_tasks_from_parquet` → `hpc.launch --job_type datagen`) are unchanged from v1.

---

## Repository Structure

Each source dataset is stored as a subdirectory named `org__name/`, where the original HuggingFace repo `org/name` has its `/` replaced with `__`. The raw Parquet shards, README, and any other files from the source repo are preserved exactly as downloaded — no extraction or transformation is applied.

For example:
- `DCAgent/swe_rebench_patched_oracle` → `DCAgent__swe_rebench_patched_oracle/`
- `laion/exp_rpt_stack-php-v2-v6` → `laion__exp_rpt_stack-php-v2-v6/`
- `SankalpKJ/swesmith-oracle-filtered` → `SankalpKJ__swesmith-oracle-filtered/`

---

## Task Format

All tasks in TaskTrove are valid [Harbor](https://github.com/open-thoughts/OpenThoughts-Agent) task binaries.

Tasks fall into two categories:

- **With verifiers** (RL training / eval): include a `verifier` field that scores agent trajectories at episode end. Examples: `swe_rebench_patched_oracle`, `r2egym-patched-full-oracle`, the `exp_rpt_*` and `nemotron-gym-*` family.
- **Without verifiers** (SFT / datagen): define the task environment and instruction but rely on external grading or teacher-model judgment. Examples: `selfinstruct-naive-sandboxes-2-verified`, `nl2bash-tasks-cleaned-oracle`.

## Relationship to AgentTrove

The task datasets in TaskTrove are the direct sources for the agent traces in **[AgentTrove](https://huggingface.co/datasets/open-thoughts/AgentTrove)**. To generate a trace dataset, a teacher model (e.g. GLM-4.7, Kimi K2.0 Thinking, GPT-5.1 Nano) is run as a Harbor agent against a task binary, producing a trajectory per task. AgentTrove aggregates ~1.7 million such trajectories across 219 such runs.

## Usage

### 1. Extract tasks from a binary

Use the extraction tool in [OpenThoughts-Agent](https://github.com/open-thoughts/OpenThoughts-Agent) to unpack a task binary into a directory of individual task files:

```bash
python -m scripts.datagen.extract_tasks_from_parquet \
  --parquet DCAgent/swe_rebench_patched_oracle \
  --output_dir $SCRATCH/tasks/swe_rebench_patched_oracle \
  --on_exist overwrite
```

The `--parquet` argument accepts either a local path or a HuggingFace dataset repo ID. The `--output_dir` will contain one JSON file per task.

### 2. Generate new agent traces

Once tasks are extracted, launch a Harbor datagen job to generate traces using any vLLM-served model:

```bash
python3 -m hpc.launch \
  --job_type datagen \
  --trace_harbor_config "./hpc/harbor_yaml/datagen/ctx32k_verified.yaml" \
  --datagen_config kimi_k2_5_vllm_serve_torch_h200.yaml \
  --tasks_input_path "$SCRATCH/tasks/swe_rebench_patched_oracle" \
  --trace_target_repo DCAgent2/Kimi-2.5-swe_rebench-sandboxes-maxeps-32k \
  --time_limit 47:59:00 \
  --num_nodes 1 \
  --gpus_per_node 8 \
  --trace-n-concurrent 20
```

Key flags:
- `--trace_harbor_config` — Harbor agent configuration (context length, summarization, tool parser, etc.)
- `--datagen_config` — vLLM serving config (model, tensor parallelism, cluster hardware)
- `--tasks_input_path` — directory of extracted task files
- `--trace_target_repo` — HuggingFace repo where output traces will be uploaded
- `--trace-n-concurrent` — number of simultaneous Harbor episodes

See the [OpenThoughts-Agent repository](https://github.com/open-thoughts/OpenThoughts-Agent) for full documentation, available Harbor configs, and cluster setup guides.

## Resolving v1

If you need the exact contents of v1 (the original ~119-subdir release), pin to the `v1` tag:

```python
from huggingface_hub import snapshot_download
snapshot_download(repo_id="open-thoughts/TaskTrove", repo_type="dataset", revision="v1", local_dir="./tasktrove_v1")
```
